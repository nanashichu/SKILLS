# 文件格式规范 (v6.3)

所有文件位于 `$TODO_DIR` 目录下。

---

## 每日日志格式（daily/YYYY-MM-DD.md）

### 区块顺序（严格固定）

```
1. 🕰️ 打卡记录
2. 🕐 时间线记录
3. 📝 经验教训（AI生成，分类展示）
4. 💭 今日感悟（AI提取，分类展示）
5. 📋 今日待办
6. ✅ 今日已完成
7. 📋 明日待办
8. 📓 今日日记（用户手写，AI绝对不碰）
```

**禁止出现：** `## 🍅 番茄统计`、`## 📊 今日统计`、`## 🌅 明日建议`

### 格式模板

```markdown
# YYYY年MM月DD日 工作日志

## 🕰️ 打卡记录

### 上午
- 上班：HH:MM
- 下班：HH:MM | 进行中
- 暂停：HH:MM-HH:MM（XX分钟）【可选】
- 有效时长：X小时X分钟

### 下午
- 上班：HH:MM
- 下班：HH:MM | 进行中
- 有效时长：X小时X分钟

### 晚上
- 上班：HH:MM
- 下班：HH:MM | 进行中
- 有效时长：X小时X分钟

**今日总工作时长**：X小时X分钟

## 🕐 时间线记录

- **HH:MM-HH:MM** [项目名] - 任务名称 | 用时：X小时

## 📝 经验教训

### 实证方法
- ✅ 做得好的：...
- ⚠️ 需改进：...
- 💡 新发现：...

### 工具使用
- ...

### 项目管理
- ...

## 💭 今日感悟

### 饮食健康
- 感悟内容

### 情绪心理
- 感悟内容

## 📋 今日待办

### 🔥 今日重点
- [ ] ⚠️ [项目名] - 任务（截止：MM-DD，还剩X天）
- [ ] [项目名] - 任务

### 其他任务
- [ ] [项目名] - 任务

## ✅ 今日已完成

- [x] 任务名称

## 📋 明日待办

### 🔥 明日重点
- [ ] 任务

### 📌 其他
- [ ] 任务

## 📓 今日日记

（用户手写区块，AI不得修改、删除或添加内容）
```

## 项目文件格式（projects/YYYY-MM-DD-项目名.md）

```markdown
# 项目：[项目名]

**创建日期**：YYYY-MM-DD
**截止日期**：YYYY-MM-DD 或 无
**优先级**：高/中/低
**状态**：活跃/暂停/完成

## 📊 项目进度
- 总任务数：X
- 已完成：X
- 进度：XX%

## 任务列表
- [✅ YYYY-MM-DD] 任务名称
- [ ] 任务名称

## 📅 项目日志
### YYYY-MM-DD
- **背景**：为什么要做
- **做了什么**：具体操作
- **结果/产出**：改变了什么

## 📂 关键文件索引
- [文件描述](相对路径)

## 经验教训
- 💡 新发现（YYYY-MM-DD）
- ⚠️ 踩坑（YYYY-MM-DD）
```

## 配置文件格式（config.md）

位于 `$TODO_DIR/config.md`，用于自定义系统行为：

```markdown
# 配置

## 固定任务
- 每日习惯任务1
- 每日习惯任务2

## 任务类型关键词
- 实证研究：回归、面板、Stata、数据、基准回归、DiD
- 编程开发：代码、开发、调试、部署
- 论文写作：写作、撰写、修改、综述
- 会议沟通：会议、交流、讨论、汇报

## 风险阈值
- 紧急截止天数：3
- 项目停滞警告天数：5
- 低效连续天数：3
```

```json
      "incompleteTasks": ["任务1", "任务2"]
    }
  },
  "fixedTasks": ["习惯任务1", "习惯任务2"]
}
```
