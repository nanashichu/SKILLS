## 跨对话搜索与日志更新（§10 核心）

### 触发条件

"结束工作"、"更新日志"、"今天干了什么"

### 搜索路径

默认扫描 `$CLAUDE_TRANSCRIPT_DIR` 下所有项目的 JSONL 对话文件。通过 `--transcript-dir` 参数可指定目录。

### 搜索流程

1. 扫描所有项目目录，用 mtime 粗筛（目标日期 ±1天）
2. 解析 JSONL，提取 `type: "user"` 消息
3. **UTC → 北京时间（+8h）**，按目标日期筛选
4. 过滤噪音：skill加载、command-message、中断信号、长度 ≤5
5. 排除当前会话 ID

### 结果分流

| 内容类型 | 写入位置 |
|---------|---------|
| 项目相关工作 | `projects/对应项目.md` 📅项目日志 |
| 工作时间线 | `daily/YYYY-MM-DD.md` 🕐时间线记录 |
| 已完成任务 | `daily/YYYY-MM-DD.md` ✅今日已完成 |
| 非工作内容 | `daily/YYYY-MM-DD.md` 💭今日感悟 |
| 新项目/新skill | 新建 projects 文件 |

### 时间线合并去重

- 先读取已有时间线
- 只补充未记录的时间段
- 按北京时间排序

### 项目关联识别

通过消息关键词、文件路径、JSONL 所在目录名判断归属。项目关键词映射在 `scripts/search_recent_sessions.py` 的 `load_project_keywords()` 中配置。

## 缓存优化机制（§7 核心）

### 缓存文件

`$TODO_DIR/memory/project-tasks-cache.json` — 所有项目的任务摘要

### 重建触发条件

- 缓存不存在
- `lastUpdated` ≠ 今天
- 缓存版本过期

### 重建流程

1. 运行 `python scripts/scan_projects.py`
2. 自动扫描 `$TODO_DIR/projects/*.md` 所有文件
3. 提取：进度、未完成任务、截止日期
4. 写入缓存，更新 `lastUpdated`

### 读取策略

生成今日待办最多 2 次读取（daily + 缓存），不逐个读项目文件。

## 数据收集机制

### 触发点

| 操作 | 写入 | 内容 |
|------|------|------|
| "现在开始 [任务]" | `memory/session-context.json` | currentTask, startTime |
| "完成了" | `memory/task-time-history.json` | task, type, actual, timeSlot |
| "结束工作" | `projects/*.md` | 项目进展同步 |

### task-time-history.json

```json
{
  "taskHistory": [
    {"date": "2026-05-20", "task": "重写机制检验代码", "type": "实证研究", "actual": 1.2, "timeSlot": "afternoon"}
  ],
  "typeAverages": {
    "实证研究": {"avgActual": 1.5, "count": 12}
  },
  "timeSlotEfficiency": {
    "morning": {"tasks": 5, "hours": 8},
    "afternoon": {"tasks": 15, "hours": 22},
    "evening": {"tasks": 10, "hours": 18}
  }
}
```

## 任务时间预测（§4 附带）

### 预测算法

```
基础预测 = 同类任务历史平均用时（从 task-time-history.json 读取）
置信区间 = [预测值 × 0.8, 预测值 × 1.3]
```

### 任务类型识别

| 类型 | 关键词（可在 config.md 中配置） |
|------|------|
| 实证研究 | 回归、面板、Stata、数据、基准回归、DiD |
| 论文写作 | 写作、撰写、修改、综述 |
| 编程开发 | 代码、开发、skill、上传 |
| 会议沟通 | 会议、交流、讨论、汇报 |

## 风险预警详情（§11）

### 截止风险

| 级别 | 条件 | 标识 |
|------|------|------|
| 极紧急 | 截止 = 今天 | 🚨 |
| 紧急 | 截止 ≤ 1天 | ⚠️ |
| 提醒 | 截止 ≤ 3天 | 📌 |
| 逾期 | 截止 < 今天 | 🔴 |

### 习惯风险

- 固定任务（在缓存 `fixedTasks` 中配置）间隔超阈值 → 提醒
- 项目停滞 > 5天无任何时间线记录 → 标记

### 效率风险

- 连续 3 天完成任务数 < 日均 50%
- 缓存中进度长期不更新

## 经验教训自动生成（§9）

### 生成流程

1. 读取今日时间线 + 已完成任务
2. 按类别分组提炼

### 分类

| 分类 | 涵盖内容 |
|------|---------|
| 实证方法 | 回归设定、检验选择、数据处理决策 |
| 工具使用 | Stata/Python/Claude Code 操作技巧、踩坑 |
| 项目管理 | 时间分配、优先级判断、协作 |
| 论文写作 | 结构、表达、逻辑 |
| 其他 | 以上不覆盖的 |

### 标记体系

- ✅ 做得好的：今天什么有效
- ⚠️ 需改进：什么可以更好
- 💡 新发现：学到的新东西

## 生活感悟提取（§8）

### 分类

| 分类 | 关键词 |
|------|--------|
| 饮食健康 | 减脂、零食、健身、睡眠、运动 |
| 情绪心理 | 不想干活、焦虑、开心、疲惫、压力 |

| 日常琐事 | 鼠标延迟、电脑问题、购物 |

### 提取方式

1. 扫描 JSONL 时标记非工作类用户消息
2. 提炼为 1-2 句感悟，按分类归组
3. 不照搬原文，提炼核心观点
