# 常见错误与陷阱 (v6.3)

---

## 🔴 Critical

### 陷阱 0：打卡记录重复生成

**错误表现：** 同一时段出现多个 "### 上午" 区块

**根本原因：** 写入前没检查已有打卡区块

**正确做法：**
1. 写入前 Read 文件，检查已有打卡时段
2. 同时段 → Edit 更新，不追加
3. 不同时段 → 追加新区块
4. 写入后验证：确认无重复时段

---

### 陷阱 1：用 Write 覆盖整个文件

**错误表现：** 时间线、感悟、已完成任务被清空

**正确做法：** 必须用 Edit 增量更新，只修改目标区块

---

### 陷阱 2：经验教训区块为空

**错误表现：** `## 📝 经验教训` 缺失或内容为空

**正确做法：** "结束工作"时必须从时间线+已完成任务提炼经验，按分类展示。无实质工作时写 "今日无实质工作经验，暂无经验教训"

---

### 陷阱 3：每日模板漂移

**错误表现：** 多出不该有的区块（🍅番茄统计、📊今日统计），缺少必需区块（📝经验教训、📓今日日记）

**根本原因：** 没有遵循标准模板

**正确做法：** 写 daily 文件前必须确认区块顺序，参考 `references/file-formats.md` 中的模板，写完后逐区块对比

---

## 🟡 数据同步

### 陷阱 4：daily 和 projects 不同步

**错误表现：** 任务完成了但项目进度没更新

**正确做法：** "结束工作"时同步所有涉及的项目文件，双向写入

---

### 陷阱 5：截止日期信息不准确

**错误表现：** 剩余天数计算错误、日期混淆

**正确做法：** 输出日期前先确认 `今天是 YYYY年MM月DD日 周X`，截止 - 今天 = 剩余天数

---

## 🟠 格式

### 陷阱 6：待办格式不正确

```
✅ - [ ] 项目名 - 任务内容
❌ - 任务内容（缺少 [ ]）
❌ [ ] 任务内容（缺少 - 前缀）
```

---

### 陷阱 7：跨对话搜索遗漏

**错误表现：** "结束工作"时只写当前会话内容，遗漏其他会话的工作

**正确做法：** 必须运行 `scripts/search_recent_sessions.py` 扫描所有 JSONL，按北京时间筛选，排除当前会话和噪音，合并去重

---

### 陷阱 8：时区转换遗漏

**错误表现：** JSONL 时间线错位（UTC vs 北京时间差8小时）

**正确做法：** JSONL timestamp 是 UTC，必须 +8h 转北京时间

---

### 陷阱 9：缓存过期未重建

**错误表现：** 今日待办显示的是昨天的任务列表

**正确做法：** 每次"开始工作"前检查 `lastUpdated`，过期则运行 `scripts/scan_projects.py`

---

## 🟢 预防检查清单

```
□ 是否已 Read 目标文件？
□ 是否使用 Edit 而非 Write？
□ 是否会覆盖已有数据？
□ 是否确认了当前日期？
□ 写 daily 前是否对照了标准模板？
□ 截止日期是否准确？
□ daily 和 projects 是否双向同步？
□ 缓存是否过期？
□ 跨对话搜索是否已运行？
```
