## §0 打卡记录管理

**触发条件：** "打卡"、"上班"、"下班"、"暂停"、"继续"

**打卡记录格式：**
```markdown
## 🕰️ 打卡记录

### 上午
- 上班：HH:MM
- 下班：HH:MM
- 暂停：HH:MM-HH:MM（XX分钟）【可选】
- 有效时长：X小时X分钟

### 下午
- 上班：HH:MM
- 下班：HH:MM | 进行中
- 有效时长：X小时X分钟

### 晚上
- 上班：HH:MM
- 下班：HH:MM
- 有效时长：X小时X分钟

**今日总工作时长**：X小时X分钟
```

**有效时长计算：** `(下班 - 上班) - 所有暂停时长之和`

**防重复规则（关键）：**
1. 写入前必须 Read 文件，检查已有时段区块
2. 同时段：更新已有数据，不追加新区块
3. 不同时段：追加新区块
4. 禁止同一时段出现两个区块
5. 每次打卡后必须验证无重复

## §1 未来日期事件快速添加

**触发条件：** "X月X日要XXX"、"X月X日有XXX"

**执行步骤：**
1. 解析日期为 `YYYY-MM-DD`（默认当年）
2. 检查 `$TODO_DIR/daily/YYYY-MM-DD.md` 是否存在
3. 不存在则创建，存在则追加到 `## 📋 今日待办`

---

## §2 创建新项目

**触发条件：** "项目：[项目名称]"

**支持元数据：** 截止日期、优先级（高/中/低）、当前阶段

**执行步骤：**
1. 在 `$TODO_DIR/projects/` 创建 `YYYY-MM-DD-项目名.md`
2. 写入元数据和空任务列表
3. 更新 `$TODO_DIR/memory/project-tasks-cache.json`

---

## §3 添加任务

**触发条件：** "添加任务"、"新任务"

**执行步骤：**
1. Read 项目文件
2. Edit 追加到 `## 任务列表`
3. 更新缓存 `$TODO_DIR/memory/project-tasks-cache.json`

---

## §4 开始任务 + 时间预测

**触发条件：** "现在开始"、"开始做"、"开始[任务名]"

**执行步骤：**
1. 读取项目文件，检查任务是否存在
2. 更新 `$TODO_DIR/memory/session-context.json`：记录 currentTask、startTime
3. 从 `$TODO_DIR/memory/task-time-history.json` 读取同类任务历史用时
4. 输出时间预测（如有历史数据）

**时间预测算法：**
```
基础预测 = 同类任务历史平均用时
置信区间 = [预测值 × 0.8, 预测值 × 1.3]
```

---

## §5 完成任务

**触发条件：** "完成了"、"做完了"、"结束[任务名]"

**执行步骤：**
1. 从 session-context.json 读取开始时间，计算实际用时
2. 写入 `$TODO_DIR/memory/task-time-history.json`（任务名、类型、用时、时段）
3. 更新 daily 文件 `## 🕐 时间线记录` 和 `## ✅ 今日已完成`
4. 更新项目文件：任务列表标记 ✅、📅项目日志追加、进度更新
5. 更新缓存 `$TODO_DIR/memory/project-tasks-cache.json`

---

## §6 临时任务管理

**活跃 vs 历史拆分：**

| 状态 | 位置 |
|------|------|
| 进行中/未完成 | `$TODO_DIR/projects/临时任务.md` → 🟡 活跃临时任务 |
| 已完成 | `$TODO_DIR/projects/临时任务.md` → ✅ 历史记录 |

## §7 生成今日待办（早上好）

**触发条件：** "早上好"、"开始工作"、"开工"

**执行步骤：**
1. Read `$TODO_DIR/daily/YYYY-MM-DD.md`
2. Read `$TODO_DIR/memory/project-tasks-cache.json`（过期则重建）
3. 检测是否有特定类型任务（关键词可在 config.md 中配置）
4. 如有特定类型任务，输出激励内容
5. 自动执行 §11 风险预警
6. 输出今日任务概览

**输出格式：**
```
🌅 早上好！今天是 YYYY年MM月DD日 周X

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[激励内容 - 如有特定类型任务]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 今日待办提醒

⚠️ 紧急任务（3天内）：
📌 今日重点：
✅ 已完成：X 个 | ⏳ 待办：X 个

💡 今日可用功能：打卡 / 开始任务 / 完成任务 / 结束工作 / 检查风险
```

## §8 自动生成感悟

**触发条件：** "生成感悟" 或 "结束工作"时自动触发

**执行步骤：**
1. 分析今日时间线记录
2. 提取用户非工作类消息（情绪/生活/饮食等）
3. 按分类归组写入 `## 💭 今日感悟`

**分类体系：** 饮食健康、情绪心理、生活习惯、日常琐事

---

## §9 经验总结

**触发条件：** "结束工作"时自动触发（强制！）

**执行步骤：**
1. 读取今日已完成任务和时间线
2. 按类别提炼经验教训：
   - 实证方法（回归、检验、数据处理）
   - 工具使用（Stata、Python、Claude Code）
   - 项目管理（时间分配、优先级）
   - 论文写作（结构、表达、逻辑）
   - 其他
3. 每类用标记：✅ 做得好 / ⚠️ 需改进 / 💡 新发现
4. 写入 `## 📝 经验教训`
5. **禁止空区块**：无实质工作时写"今日无实质工作经验"

## §10 每日总结 + 跨对话搜索

**触发条件：** "结束工作"、"更新日志"

**执行步骤：**
1. 执行跨对话搜索（扫描 `$CLAUDE_TRANSCRIPT_DIR` 下所有 JSONL）
2. 按北京时间（UTC+8）筛选今日用户消息
3. 排除当前会话、skill加载、command-message 等噪音
4. 结果分流：
   - 工作内容 → daily 时间线 + projects 项目日志
   - 已完成任务 → daily 已完成 + projects 历史记录
   - 非工作内容 → daily 感悟
5. 时间线与已有记录合并去重
6. 同步更新所有涉及的 projects/*.md
7. 自动触发 §8 感悟 + §9 经验

**⚠️ 日志详细程度标准（写入前强制执行）：**

每天每条日志必须包含五要素：**目的 + 对象 + 操作 + 数据/文件 + 结果**。

- **每日时间线**：写完后一个陌生人只读这条，能否复现你的操作？必须能。
- **项目日志**：三段式——背景/目的、做了什么、结果/产出。

**自检：** 写完每条问自己——(1)为什么做？(2)在论文哪部分？(3)能定位到对应数据/代码吗？(4)能复现出同样结果吗？四个必须全为是。

## §11 风险预警+Guard

**触发条件：** 每日首次对话/开始工作时自动 + "检查风险"手动

**预警维度：**

| 维度 | 检测内容 |
|------|---------|
| 截止风险 | 3天内截止 / 已逾期任务 / 逾期超7天 |
| 习惯风险 | 连续N天未完成习惯任务 / 项目停滞超5天 |
| 效率风险 | 连续3天低效 / 任务完成率低于日均50% |
| 缓存过期 | `project-tasks-cache.json` lastUpdated ≠ 今天 |

Guard 配置（`$TODO_DIR/memory/guard-config.json`）支持自定义阈值和级别。
