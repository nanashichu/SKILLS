# 提炼规则详细说明

## 五条核心原则

### 1. 去过程留结果

**保留**：结论、决策、发现、踩坑终点
**丢弃**：中间调试步骤、ls/cat 输出、反复试错的过程

示例：
- ❌ "先 ls 看了目录，发现文件在子目录下，又 cd 进去，cat 了一下发现格式不对"
- ✅ "文件实际位于子目录下，格式为 CSV 而非预期的 XLSX"

### 2. 去重复留精华

同一问题反复调试 → 压缩为最终方案。

示例：
- ❌ 记录三次尝试 merge 失败的详细过程
- ✅ "merge 前必须检查 key variable 类型一致性（string vs double 会导致静默匹配失败）"

### 3. 保留决策逻辑

不只记「做了什么」，要记「为什么这么做」。

示例：
- ❌ "用了 pair FE"
- ✅ "固定效应选择用 pair FE，因为配对层面存在不可观测的异质性"

### 4. 保留个人反思

用户的自我反思、踩坑感悟、工作节奏反思，这些比技术笔记更有价值。

识别关键词：
- "我觉得"、"我认为"、"说实话"
- "不能"、"必须"、"千万不要"
- "上瘾"、"浪费时间"、"零进展"
- 情绪表达（"无语"、"离谱"、"太爽了"）

### 5. 研究内容脱敏

**不写**：具体研究话题名称、具体变量名、具体数据来源名称
**可以写**：方法论、工具使用、踩坑经验、通用决策逻辑

| 原始内容 | 脱敏后 |
|---------|--------|
| "数智化转型与股价崩盘风险的VaR计算" | "某实证项目的风险度量方法" |
| "知识产权法庭与科技创业" | "某政策评估项目" |
| "供应链溢出的控制变量" | "某溢出效应研究的变量选择" |
| "杨金玉李青原陶锋三篇顶刊" | "三篇相关顶刊文献" |
| "stata-mcp 太慢改用 python" | ✅ 直接写（工具类不脱敏） |
| "skill 开发原则" | ✅ 直接写 |
| "merge 键类型一致性" | ✅ 直接写（方法论不脱敏） |

## 内容分类规则

### 实证研究方法论 → `实证研究/`

触发关键词：回归、面板、DID、固定效应、控制变量、机制检验、稳健性、内生性、工具变量、分位数回归、基准回归

笔记内容：
- 识别策略选择及理由
- 变量构建方法及理由
- 固定效应选择及理由
- 机制检验路径设计
- 稳健性检验清单
- 审稿意见应对方法论

### 数据处理踩坑 → `数据资源/`

触发关键词：merge、数据合并、类型不一致、编码、缺失值、清洗、格式转换、stata-mcp、python 读取

笔记内容：
- 数据合并踩坑（key variable 类型、编码问题）
- 工具选择决策（stata-mcp vs python）
- 数据格式转换经验
- 大文件处理策略

### Claude Code 工具链 → `工具资源/`

触发关键词：skill、CLAUDE.md、memory、飞书、stata-mcp、agent、hook、配置、vibe-coding

笔记内容：
- Skill 开发原则和踩坑
- 记忆管理经验
- 工具配置经验
- 多 Agent 协作经验
- vibe-coding 产出记录

### 个人反思 → `生活杂记/`

触发关键词：我觉得、反思、上瘾、效率、节奏、不能替代、编造、错误

笔记内容：
- AI 协作反思
- 工作节奏反思
- 严重错误记录
- 工具 vs 实证平衡反思

## 笔记格式模板

### 新建笔记

```markdown
# 主题 — 从对话中提炼

> 从历史对话（YYYY-MM-DD ~ YYYY-MM-DD）提炼

## 子主题1

- 结论/决策
- 决策理由

## 子主题2

- 踩坑记录
- 最终方案

## 个人反思（如有）

> 引用原话 — 日期

反思内容
```

### 追加到已有笔记

在文件末尾追加新日期的内容，用 `## YYYY-MM-DD 补充` 分隔：

```markdown
## YYYY-MM-DD 补充

### 新发现的子主题

- 新结论/决策
- 新踩坑

### 更新已有认知

- 之前认为 X，现在发现 Y
```
